# CI Reports

A GitHub Pages site that lists and hosts test reports generated by your CI pipelines.

## How it works

```
Your CI repo                       This repo (uitest-report)
─────────────                      ─────────────────────────
 run tests
 generate report.html  ──push──►  reports/YYYY-MM-DD/report.html
                                   reports.json (updated)
                                        │
                                   GitHub Pages deploys
                                        │
                                   https://<owner>.github.io/uitest-report/
```

## Repository structure

```
uitest-report/
├── index.html               # Landing page (auto-loads reports.json)
├── reports.json             # Report manifest — updated by CI
├── reports/                 # Report files committed by CI
│   └── YYYY-MM-DD/
│       └── report.html
├── scripts/
│   └── add-report.sh        # CI helper script
└── .github/workflows/
    ├── pages.yml            # Deploy this site to GitHub Pages
    └── example-ci.yml      # Example workflow for your project CI
```

## Setup

### 1. Enable GitHub Pages

In this repository: **Settings → Pages → Source → GitHub Actions**

### 2. Add a report from CI

In your project's CI workflow, clone this repo and call `add-report.sh`:

```yaml
- name: Publish report
  env:
    GITHUB_TOKEN: ${{ secrets.REPORTS_PUSH_TOKEN }}
  run: |
    git clone "https://x-access-token:${GITHUB_TOKEN}@github.com/YOUR_ORG/uitest-report.git" _reports_site
    cd _reports_site
    bash scripts/add-report.sh ../path/to/report.html "Build #${{ github.run_number }}"
```

> **Tip:** Create a PAT with `repo` scope (or a fine-grained token with Contents: Read & Write)
> and store it as secret `REPORTS_PUSH_TOKEN` in your project repository.

### 3. Full example

See [`.github/workflows/example-ci.yml`](.github/workflows/example-ci.yml) for a complete workflow you can copy into your project.

## Script reference — `scripts/add-report.sh`

```
Usage: ./scripts/add-report.sh <report-file> <report-name>

Arguments:
  report-file   Path to the generated report (HTML, XML, etc.)
  report-name   Label shown on the landing page

Optional env vars:
  REPORT_DATE     Override date (YYYY-MM-DD). Default: today UTC.
  GIT_USER_NAME   Commit author name.  Default: "CI Bot"
  GIT_USER_EMAIL  Commit author email. Default: "ci@noreply"
  SKIP_GIT_PUSH   Set to "1" to skip git push (dry-run).
```

## Landing page features

- Reports grouped by date, newest first
- Pagination — 10 date-groups per page
- Each group shows unlimited reports
- Clicking a report opens it in a new tab
- Works as a plain static site — no build step needed
